NOTES

What to preserve, what to polish

Preserve: The dark theme, the brand gradient, DM Sans + Playfair Display + JetBrains Mono combination, the 3-level text hierarchy, the subtle borders. These are the identity.
Priority polish zones:
1. The AI Moment sequence (B3 → B4 → B5) — this is demo day. Every frame-to-frame transition must feel cinematic.
2. Customization Hub (C3) — currently functional but dense. Needs better visual grouping and a proper preview mode.
3. Public site hero (D1) — this is what customers see first. Must feel premium, not generic.
4. Booking drawer (D2) — currently works but looks utilitarian. Needs warmth and confidence.
Do not change:
• The underlying booking state machine (pending → confirmed → completed → cancelled)
• The subdomain routing logic
• The auth flow and RLS policies
• The Supabase schema (columns that already exist)
• Next.js 14.2.29 version
Open design questions:
• Should the success overlay (B5) auto-dismiss after 1.8s or require explicit user action?
• Should variant preview cards show more than just the hero, or is the hero enough to convey the vibe?
• Is the booking drawer right-sided, or should it be a center modal on mobile?
• What's the empty state for the dashboard when a business has zero bookings yet?
